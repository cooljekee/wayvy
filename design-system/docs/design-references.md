# Wayvy — Design References

**Date:** 2026-05-15

Референсы дизайн-систем и UI-паттернов популярных приложений, работающих с картами. Собраны как основа для дизайн-системы Wayvy.

---

## 1. Google Maps

**Почему релевантен:** лидер рынка картографии, задаёт стандарты UX для карт на iOS/Android.

### Ключевые принципы
- **Material 3 Expressive** — воздушный, минималистичный вид; информация организована иерархически, не перегружена
- **Modernized pins/markers** (с 2024): улучшенная читаемость, консистентность, accessibility; чёткое различие между типами POI по форме и цвету
- **Палитра:** прохладный синий акцент (обновление март 2025), нейтральные поверхности; карта остаётся «фоном», UI-хром не конкурирует с контентом
- **Places UI Kit** — переиспользуемые компоненты для POI: фото, рейтинг, часы работы, действия (маршрут, позвонить, сайт)

### Применимо к Wayvy
- Пины трёх типов: свой маршрут / подписка / событие — различаются формой, не только цветом
- UI-хром (карточки, кнопки) не должен перекрывать карту — использовать прозрачность/blur
- Акцент на accessibility: цвета + форма, не только цвет

### Ресурсы
- [Google Maps UI Views for Mobile 2024 — Figma Community](https://www.figma.com/community/file/1352319963071516110/google-maps-ui-views-for-mobile-2024)
- [Designing Google Maps — Medium](https://medium.com/@jyoti1308/designing-google-maps-d9865c3506ba)

## 2. Apple Maps (Human Interface Guidelines)

**Почему релевантен:** нативный iOS, задаёт паттерны взаимодействия которые пользователи уже знают.

### Ключевые принципы
- **Минимализм:** карта — главный элемент, весь UI-хром убирается в sheet или overlay
- **Bottom Sheet как основной паттерн:** поиск, детали места, маршрут — всё в draggable sheet (3 состояния: collapsed / half / expanded)
- **SF Symbols:** нативная иконография, масштабируется с Dynamic Type, работает с tint-цветом
- **Плавающие элементы** над картой: кнопка геолокации, компас, слои — строго top-right или bottom-right, не перекрывают маршрут

### Применимо к Wayvy
- RecordingBar — floating HUD, прижат к bottom safe area
- WaypointModal — bottom sheet с тремя состояниями
- MapControls — геолокация, слои — правый угол, не левый

### Ресурсы
- [Apple HIG — Maps](https://developer.apple.com/design/human-interface-guidelines/maps)

## 3. Strava

**Почему релевантен:** ближайший аналог Wayvy — GPS-запись активностей + социальный граф.

### Ключевые принципы
- **Bold, athlete-first:** крупный текст (читается в движении), крупные кнопки (нажимаются на ходу), высокий контраст
- **Оранжевый акцент (#FC4C02)** — узнаваемый, энергичный, хорошо виден на светлом и тёмном фоне
- **Социальный фид:** full-width фото активности + inset-карта маршрута (не наоборот)
- **Карточка активности:** превью карты маршрута в виде полилинии, метрики (дистанция, время, темп), автор
- **Сегменты и KOM** — геймификация через цвет (gold, purple) прямо на карте

### Применимо к Wayvy
- RouteCard: превью полилинии + ключевые метрики (дистанция, время, город)
- Акцент-цвет должен быть энергичным и читаться в движении
- Кнопки в RecordingBar: минимум 44pt touch target, высокий контраст

## 4. AllTrails

**Почему релевантен:** GPS-маршруты + точки интереса + фото + социальные элементы — структура очень близка к Wayvy.

### Ключевые принципы
- **8pt grid** — вся сетка и компоненты кратны 8
- **Зелёный как природный акцент:** ассоциация с активным отдыхом на природе
- **Style guide:** централизованные компоненты, цвета, типографика, иконки — единая библиотека
- **Trail Card:** фото + рейтинг + дистанция + набор высоты + сложность + количество отзывов
- **Карта + список** — два режима просмотра маршрутов; активное переключение без потери контекста

### Применимо к Wayvy
- Строгий 8pt grid для всего
- RouteCard: обязательно превью + метрики + автор, без перегрузки
- Дуальный режим «карта / список» на экране Профиля и Событий

## 5. Komoot

**Почему релевантен:** аутдор-навигация с акцентом на маршруты и места; близкий визуальный тон к Wayvy.

### Ключевые принципы
- **Качество бумажной карты на экране:** детализация, читаемость troп, рельеф — карта несёт смысловую нагрузку, не просто фон
- **Нейтральная палитра:** серый + чёрный + тёмно-зелёный + светло-зелёный — работает без перегрузки на фоне насыщенной карты
- **Спорт-специфичные карты:** цвет слоёв меняется под контекст (хайкинг / байкинг / бег)
- **Минимальный хром:** интерфейс уходит с дороги, пока пользователь движется

### Применимо к Wayvy
- Цветовые токены для полилиний маршрута: свой маршрут vs маршрут подписки — разные оттенки одного акцента, не две несвязанные краски
- UI-хром в режиме записи: максимально убирать, оставлять только критичное

## 6. Citymapper

**Почему релевантен:** сильный brand identity вокруг одного акцент-цвета + городской контекст.

### Ключевые принципы
- **Signature green (#00D26A)** — единственный акцент, всё остальное нейтральное; instantly узнаваем
- **Типографика:** современный sans-serif, консистентное использование weight для иерархии
- **Иконография:** clean arrow system — навигационные иконки простые, без лишних деталей

### Применимо к Wayvy
- Один сильный акцент-цвет лучше двух; поддержка нейтральными поверхностями
- Иконки в TabBar: простые, 1-2 элемента, узнаваемые в 24pt

## 7. Waze

**Почему релевантен:** высоконасыщенный realtime UI поверх карты — крайний пример, от которого Wayvy должен держаться дальше.

### Ключевые принципы
- **Bold alerts:** сильные контрастные цвета для предупреждений — работает, но перегружает
- **Кастомные пины-бренды** на карте: raises awareness, но создаёт визуальный шум

### Применимо к Wayvy (чего избегать)
- Не перегружать карту пинами; кластеризация при zoom-out обязательна
- Не использовать слишком много цветов для разных типов объектов — max 3-4 цвета на карте

## Тренды 2025–2026 (map-centric apps)

- **Glassmorphism для overlay** — blur под временными слоями (sheet, HUD), не для основных поверхностей
- **FAB → Bottom Bar** — плавающие кнопки уходят в нижнюю навигацию; исключение — RecordingBar (контекстный HUD)
- **Adaptive dark mode** — автопереключение день/ночь; карта тоже меняет тайлы (Yandex поддерживает dark scheme)
- **Gesture-first** — свайп up/down для sheet, pinch для карты, hold для добавления точки
